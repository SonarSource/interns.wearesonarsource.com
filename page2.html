Languages Team üî£

They build state-of-the-art static analyzers used by millions of developers

Technical strategy

state-of-the-art analyzers require advanced capabilities suhc as:

  • Semantic analysis
  • Control flow analysis
  • Data flow analysis
  • Cross-procedural analysis
  • Cross-file analysis
  • Type inference
  • ...

They want to mutualize tools, libraries and techniques across languages to avoid wasting their efforts and to porvide the most advanced capabilities.
In particular, they may use intermediate rpresentation which are consumed by specialized analyzers.

They aim at reusing existing parsers / front ends if they are open source and potentially contribute back.

Mission

They want to help developers to write better and sefer code by spotting issues of several kinds for all major programming languages:

  • Maintainability issues: code smells
  • Reliability issues: bugs
  • Security issues: vulnerabilities and security hotspots
